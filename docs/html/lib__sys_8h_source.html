284// Generally speaking (the order does not really matter on this):
285// 1. Set the mode bits in the master control register (GRAPHICS + BITMAP), as well as the resolution.
286// 2. Load the color table for the bitmap into one of the graphics LUTs (start at $B4:2000 on the A2560U, $FEC8:2000 on the A2560K)
287// 3. Load the bitmap pixel data into video RAM (starts at $C0:0000 on the A2560U, $0080:0000 on the A2560K)
288// 4. Set the enable bit and LUT # in the bitmap control register for the layer you want
289// 5. Set the address of the pixel data in video RAM in the VRAM address pointer for the layer you want. This must be set relative to the beginning of VRAM, so if you're using the A2560U and the pixel data starts at $C2:0000, you'd store $02:0000 in the address register.

291// TextInitialise:
292// clr.l d0
293// lea vicky3,a0 ; start register writes
294// move.l #$0000000C,(a0) ; graphics mode & bitmap on.
295// move.l d0,$4(a0) ; zero border size/scroll/colour
296// move.l d0,$8(a0) ; clear border colour
297// move.l d0,$C(a0)
298// move.l d0,$10(a0) ; cursor off
299// move.l d0,$14(a0) ; no line interrupts

303// move.l d0,$104(a0) ; BMP 1 address
